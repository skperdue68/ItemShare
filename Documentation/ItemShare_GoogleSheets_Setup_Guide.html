ItemShare Google Sheets Setup Guide

This guide explains how to set up the companion Google Sheet for ItemShare, install the Apps Script files, authorize the script the first time, and import your ItemShare saved variables file.

Before You Start

If someone already gave you a working Google Sheet, you may not need to do the full setup. In that case, you may only need to approve the script the first time you use it.

In the current package, the Google Sheets files are not linked separately in the document. Instead, the required files are included inside the ZIP file in a subdirectory named GoogleSheets. You will use the files from that folder:
  • GoogleSheets/Code.gs
  • GoogleSheets/UploadDialog.html
Quick Start Option (Recommended):
Instead of creating a new spreadsheet and manually setting everything up, you can use a pre-configured template:

Open ItemShare Template Spreadsheet

Click Use Template to create your own copy with the required structure already in place.
Sample Data Included:
The ItemShare package also includes a SampleData directory containing example SavedVariables files.

You can upload these sample files using the updater to see how the spreadsheet behaves before using your own data.

Part 1: Create the Spreadsheet Script Project

Step 1. Open Google Sheets and create a new spreadsheet. Give it a name you want to use for your ItemShare tracker.
Step 2. In the spreadsheet, open Extensions → Apps Script.
Step 3. In the Apps Script editor, open the default Code.gs file, remove the sample code Google created, and replace it with the contents of GoogleSheets/Code.gs from the ZIP package.
Step 4. In the Apps Script editor, click the + button next to files and add a new HTML file.
Step 5. Name the new HTML file UploadDialog.
Step 6. Remove the default HTML Google creates and replace it with the contents of GoogleSheets/UploadDialog.html from the ZIP package.
Step 7. Save the script project.

Part 2: Share the Spreadsheet

Once the script files are in place, configure access to the spreadsheet the same way you would for any shared Google Sheet.

  • You can share it directly with specific people.
  • Or you can allow access to anyone with the link, depending on how you want your group to use it.

Part 3: First-Time Authorization

The first time you run the updater on a new spreadsheet, Google will require authorization for the attached script.

Step 1. In the spreadsheet, use the custom Item Share menu and choose Open Updater.
Step 2. When Google shows the authorization prompt, click OK.
Step 3. If Google displays the unverified app warning, click Advanced.
Step 4. Continue to the project anyway if you trust the sheet and know it came from your ItemShare package.
Step 5. Grant the requested permissions so the spreadsheet can update itself and display the upload dialog.
You normally only need to go through these extra authorization steps once per spreadsheet. If you create a brand new spreadsheet later, you will need to authorize that new one as well.

Part 4: Import Your ItemShare Data

Step 1. In ESO, make sure you have marked the items you want to share.
Step 2. Run /ishare save or reload the UI so the latest addon data is written out.
Step 3. Return to the spreadsheet and open the updater from the Item Share menu.
Step 4. Click Choose File and browse to your ItemShare saved variables file in your ESO SavedVariables folder.
Step 5. Click Update Sheet.
Step 6. Wait for the updater to read the file and generate or refresh the sheets.

Where to Find the Saved Variables File

The updater expects the ItemShare saved variables file from your ESO installation. The PDF notes that it is typically found in the SavedVariables directory one level above the AddOns folder.

Suggested Windows path from the original guide:
Documents\Elder Scrolls Online\live\SavedVariables\ItemShare.lua

If your installation differs, use the matching ESO server folder such as live, liveeu, or the environment you actually use.

What the Spreadsheet Creates

  • A worksheet for each source account found in the imported data
  • A master sheet that combines all imported items
  • Columns for details such as name, item type, quality, trait, date added, count, requested by, and source sheet

The spreadsheet is designed so users can either browse the master sheet or look directly at a specific player's sheet.

Using the Finished Sheet

  1. Look through the Master sheet or an individual account sheet.
  2. When you want an item, enter your user ID in the Requested By column.
  3. The spreadsheet script keeps the request synchronized to the corresponding sheet entry.
  4. Once an item has been fulfilled, the provider should remove or update that entry as part of the normal workflow.

Recommended Tips

  • Keep the spreadsheet in a shared location your group can access.
  • Always save or reload the ESO UI before uploading a new file so the latest data is available.
  • If you make major script changes, save the Apps Script project before testing again.
  • If the custom menu does not appear immediately after setup, refresh the spreadsheet tab.
